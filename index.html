    Introduction

    This is API documentation for Qlue Gopay POC.

    Authentication

    To authorize, use this code:

    curl "api_endpoint_here"
  -H "Authorization: Bearer [your_token_here]"

    const axios = require('axios');

axios.get('api_localhost', {
  headers: {
    Authorization: Bearer [token]
  }
})
.then(resp => console.log(resp.data))
.catch(err => console.log(err.message))

    Pastikan [your_token_here] adalah API key dari web qlue.

    Untuk authentikasi, akan diberikan API KEY dari qlue dan dimasukkan ke dalam header. Request API bisa diambil di web.

    Jangan lupa ganti [your_token_here] dengan API key yang didapat dari web qlue

    Check In

    Jika plat nomor terdaftar maka responsenya seperti dibawah ini

    {
  "status": true,
  "message": "OK",
  "result": "Registered, Gate Open"
}

    api end point untuk checkin nodeflux

    Jika plat nomor udah check in, responsenya 

    {
  "status": false,
  "message": "Not Found",
  "result": "Already checked in"
}

    Jika plat nomor tidak terdaftar, responsenya

    {
  "status": false,
  "message": "Not Found",
  "result": "Plate number is not Registered, Open Gate"
}

    Http Request

    POST http://{{ qlue-cloud-uri }}/api/customer/check_in

    Body Parameter

    Parameter Required type Description
    plate_number true string gopay id from gopay server
    image_checkin true base64 plate number in base64 form
    vehicle true string vehicle type
    check_in true date check in timestamp

    Check Out

    Jika plat nomor terdaftar dan belum check out, maka responsenya seperti dibawah ini

    {
  "status": true,
  "message": "OK",
  "result": "Registered, Gate Open"
}

    Jika plat nomor belum check out, responsenya 

    {
  "status": false,
  "message": "Not Found",
  "result": "Already checked out"
}

    Jika plat nomor tidak terdaftar, responsenya

    {
  "status": false,
  "message": "Not Found",
  "result": "Plate number is not Registered, Open Gate"
}

    Description

    API Doc for check out nodeflux

    Http Request

    PUT http://{{ qlue-cloud-uri }}/api/customer/check_out

    Body Parameter

    Parameter Required type Description
    plate_number true string gopay id from gopay server
    image_checkout true base64 checkout image in base64 form
    vehicle true string vehicle type
    check_out true date check in timestamp

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files...just save them to the `includes` folder and add them to the top of your `index.md`'s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
